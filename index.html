So we made a helper to write poems! Heureka Science Center came to my supervisor Hannu Toivonen with the idea of an AI poetry exhibit. Hannu had been working on computational poetry for years, so it was a natural fit. The goal was to make poetry writing easy and fun for museum visitors — people with no poetry experience. The system generates candidate lines and the user just picks one — no blank page anxiety. This was a project I worked on at the University of Helsinki together with colleagues.

How the system works

The user starts by typing in some keywords — things they want the poem to be about. The system then generates several candidate lines for the first line of the poem. The user picks the one they like best. From there, the system generates candidates for the next line, based on what was already written. The user picks again — and this repeats until they're happy with their poem.

Fine-tuning mBART

mBART – multilingual sequence-to-sequence model (Liu et al., 2020)
Pre-trained on 25 languages; fine-tuned on poem corpora:

  • Finnish – Gutenberg, Wikisource, Poesia
  • Swedish – Wikisource
  • English – Gutenberg

Three model variants:

  • First-line: generates first line from keywords
  • NL-Single: next line from the previous line
  • NL-Multi: next line from up to three previous lines
We used mBART — a multilingual sequence-to-sequence model pre-trained on 25 languages, which made it a natural fit for a trilingual exhibit. An important constraint was that Heureka wanted the system to run on their own hardware — no external APIs, no ongoing costs, no changes needed after deployment. We fine-tuned it on poetry corpora in Finnish, Swedish, and English. We trained three variants: one for generating the first line from keywords, and two for generating the next line — one using just the previous line, and one using up to three previous lines.

Training examples

Model Source Target
First-line time elder Arise, as in that elder time,
sublime energetic Warm, energetic, chaste, sublime!
godlike wonder Thy wonders, in that godlike age,
NL-Single Arise, as in that elder time, Warm, energetic, chaste, sublime!
Warm, energetic, chaste, sublime! Thy wonders, in that godlike age,
NL-Multi Arise, as in that elder time, <SEP> Warm, energetic, chaste, sublime!
Arise, as in that elder time, <SEP> Warm, energetic, chaste, sublime! <SEP> Thy wonders, in that godlike age,
This slide shows how we constructed the training examples for each model variant. For the first-line model, the input is a set of keywords extracted from the target line, and the output is the line itself. For NL-Single, the input is the previous line and the output is the next one. For NL-Multi, multiple previous lines are concatenated with a special separator token, and the output is the next line coming after all the lines in the input.
